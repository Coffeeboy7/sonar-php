Shadowing fields with a local variable or with a function parameter is a bad practice reducing code readability: It makes it confusing to know whether the field or the variable is and should be accessed.

Noncompliant Code Example

class Foo {
  public $myField;

  public function doSomething() {
    $myField = 0;
    ...
  }

  public function doSomethingElse($myField) {
    ...
  }
}

Compliant Solution

class Foo {
  public $myField;

  public function Foo($myField) {
    $this->myField = $myField;
  }

  public static function build($myField) {
    ...
  }

  public function setMyField($myField){
    $this->myField = $myField;
  }
}
