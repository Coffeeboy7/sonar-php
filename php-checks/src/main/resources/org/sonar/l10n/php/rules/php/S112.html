Using such generic exceptions as Error, RuntimeException, Throwable, and Exception prevents calling methods from handling true, system-generated exceptions differently than application-generated errors.

Instead, either throw a more derived type that already exists in the language (SPL exceptions), or create your own type that derives from Exception.

Noncompliant Code Example

public void foo(String bar) throws Throwable {  // Non-Compliant
  throw new RuntimeException("My Message");     // Non-Compliant
}

Compliant Solution

public void foo(String bar) {
  if (true) {
    throw new MyOwnRuntimeException("My Message");

  // or
  } else {
    throw new UnexpectedValueException();
  }
}
